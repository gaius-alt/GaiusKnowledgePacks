SkyrimNet Bulk Importer v1.2

Pick a folder full of .sknpack files. Imports them all in one go, with optional safe replacement of existing packs that share a name.

Why this exists

The SkyrimNet web UI’s Import Pack button takes one file at a time. For a 45-file bundle, that’s 45 clicks plus a separate confirmation each. This tool reduces it to one click, one folder pick.

Install

Option 1 — Bookmarklet (no extension required)

Click the green button below. It opens bookmarklet.txt in a new tab. Copy that file’s contents and paste them into a new bookmark, following the steps below.

Open Bookmarklet URL

Add the URL as a bookmark:
  1. Click the green button. A new tab opens with the bookmarklet URL inside.
  2. In that new tab, click anywhere in the text. Press Ctrl+A to select all. Press Ctrl+C to copy.
  3. Switch back to this tab. If your bookmarks bar isn’t visible, press Ctrl+Shift+B to show it (the strip just below the address bar).
  4. Right-click an empty spot on the bookmarks bar.
  5. Choose Add favorite (Edge) / Add page (Chrome) / New Bookmark (Firefox).
  6. In the Name field, type something memorable — e.g. Bulk Import.
  7. Click in the URL field. Delete anything already there. Press Ctrl+V to paste the bookmarklet URL.
  8. Click Save. The bookmark now sits on your bookmarks bar.

To use it:

  1. Open the SkyrimNet web UI at http://localhost:8080 on the Knowledge Packs page.
  2. Click your new Bulk Import bookmark.
  3. Pick the folder containing your .sknpack files.
  4. Answer the replace-existing prompt (see below).
  5. Wait. Progress overlay top-right shows each file as it imports.

Option 2 — Tampermonkey userscript (recommended for repeat use)

Adds a permanent Bulk Import button next to the existing Import Pack button on every page load.

Modern browsers block automatic userscript installs. If you try to open the .user.js file directly, you may see a popup that says “We can’t add apps, extensions, or user scripts from this website.” That’s normal. Tampermonkey can still install the script — you just have to paste it in manually.
  1. Install Tampermonkey first. Go to tampermonkey.net and click the install button for your browser. After it’s installed, look at the top right of your browser window. You should see a small icon shaped like two black-and-red squares. If you don’t see it, click the puzzle-piece icon (also top right) and pin Tampermonkey to your toolbar.
  2. Open the Tampermonkey Dashboard. Click the Tampermonkey icon. A small menu pops up. Click Dashboard. A new tab opens.
  3. Open the new-script editor. At the top of the Dashboard is a row of tabs. Find the + tab and click it. The page switches to a code editor with placeholder text. Ignore the placeholder.
  4. Clear the editor. Click anywhere in the code area. Press Ctrl+A to highlight everything. Press Delete. The editor is empty.
  5. Copy the script. Open skyrimnet-bulk-importer.user.js:
    • On GitHub: click the file, then click the “Copy raw file” icon (two overlapping pages) in the top right of the file viewer.
    • If you downloaded the ZIP: right-click the file, choose Open with → Notepad (Windows) or TextEdit (Mac). Click in the text, press Ctrl+A then Ctrl+C.
  6. Paste into the editor. Switch to the Tampermonkey editor tab. Click inside the editor. Press Ctrl+V. The script appears — it starts with // ==UserScript==.
  7. Save. Press Ctrl+S, or click File → Save.
  8. Test. Reload the SkyrimNet web UI tab. A green Bulk Import button should appear next to Import Pack.
Button still not showing? Open the Tampermonkey Dashboard. “SkyrimNet Bulk Pack Importer” should appear in your installed-scripts list with a blue toggle switched on.

How it works

The replace-existing prompt

After picking the folder, a confirm dialog appears:

Found N .sknpack files.

REPLACE MODE: click OK ONLY IF you want any existing pack with the
same name DELETED before the new one is imported. This is destructive.

Click CANCEL to import safely (duplicates get "(1)" suffix).
Recommended for first install.

Cancel (default, safe): all files import as new packs. If a name collides with an existing pack, the new one gets a (1) suffix — nothing is deleted.

OK (Replace mode): a dry-run dialog appears next, listing every pack that would be deleted and re-imported. You confirm again before anything is actually deleted. For each file in scope:

  1. Read the pack name field from the .sknpack JSON.
  2. Search the page for a heading element (h1–h6) whose text matches exactly.
  3. Walk up the DOM looking for the smallest ancestor that contains exactly one Delete button. If no ancestor matches uniquely, abort the delete for this entry — the file imports as a duplicate.
  4. Temporarily override window.confirm to auto-accept SkyrimNet’s delete dialog, click Delete, then restore window.confirm.
  5. Wait 1.5 s for the page to refresh, then import the new .sknpack.

Progress counter

CounterMeaning
✓ importedFiles successfully sent to the server.
↻ replacedFiles where an existing same-named pack was deleted first (Replace mode only).
✗ failedFiles that errored. Check DevTools console (F12) for the underlying message.

Troubleshooting

  • Firefox — not supported. showDirectoryPicker isn’t there yet. Use Chrome or Edge.
  • “Import Pack button not found” — the SkyrimNet UI changed its DOM layout. The bookmarklet/userscript selectors will need a tweak.
  • Replace mode silently skipped a pack — the pack name in your .sknpack JSON doesn’t match the existing pack name exactly. Capitalisation, prefix differences (_VKP - ), or trailing whitespace will cause a name mismatch. Fix the JSON or rename the existing pack in the UI first.
  • Imports are slow / timing out — edit the userscript and bump DELAY_IMPORT from 2500 ms to 4000 or 5000.
