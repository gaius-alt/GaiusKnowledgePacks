SkyrimNet Bulk Importer v1.1

Pick a folder full of .sknpack files. Imports them all in one go, with optional replacement of existing packs that share a name.

Why this exists

The SkyrimNet web UI's Import Pack button takes one file at a time. For a release like the Vanilla Knowledge Packs bundle (45 files), that’s 45 clicks plus a separate confirmation each. This tool reduces it to one click, one folder pick.

Install

Option 1 — Bookmarklet

Drag this link to your bookmarks bar:

Bulk Import .sknpack

  1. Drag the green button above onto your browser’s bookmarks bar.
  2. Open the SkyrimNet web UI at http://localhost:8080, navigate to the Knowledge Packs page.
  3. Click the bookmark.
  4. Pick the folder containing your .sknpack files.
  5. Answer the replace-existing prompt (see below).
  6. Wait. Progress overlay top-right shows each file as it imports.
Drag doesn’t work? Right-click bookmarks bar → Add page… → paste the URL from bookmarklet.txt into the URL field. Name it whatever you like.

Option 2 — Tampermonkey userscript (recommended for repeat use)

Adds a permanent Bulk Import button next to the existing Import Pack button on every page load.

  1. Install Tampermonkey (Chrome / Edge / Firefox) or Violentmonkey.
  2. Open skyrimnet-bulk-importer.user.js in your browser. Your userscript manager will offer to install.
  3. Reload the SkyrimNet web UI. A green Bulk Import button now sits beside Import Pack.

How it works

The replace-existing prompt

Right after you pick the folder, a confirm dialog appears:

Found N .sknpack files.

OK     = Replace any existing pack with the same name (delete old, import new)
Cancel = Add as new packs (duplicates get a "(1)" suffix)

OK — for every file, the importer:

  1. Reads the pack name field from inside the .sknpack JSON (matches what the web UI displays).
  2. Scans the page for a pack card whose visible title matches that name.
  3. If found, locates that card’s Delete button.
  4. Temporarily overrides window.confirm to auto-accept the deletion dialog, clicks Delete, then restores window.confirm.
  5. Waits 1.5 seconds for the page to refresh, then imports the new .sknpack.

Cancel — the importer skips the delete step entirely and just imports each file. Packs with names that already exist will be created as Pack Name (1), Pack Name (2), and so on. Original behavior of v1.0.

OK mode is destructive. Any pack with a matching name is deleted, along with all of its entries, before the new version is imported. If you have hand-edited entries in an existing pack, export it first or use Cancel mode and merge manually.

What the progress overlay shows

CounterMeaning
✓ importedFiles that successfully made it to the server.
↻ replacedFiles where an existing pack with the same name was deleted first. Counts the deletion step, not the import — a replaced file is also counted as imported.
✗ failedFiles that errored. Check DevTools console (F12) for the underlying message.

What happens if a file is malformed

The file is fed to the same input the manual Import Pack button uses. The SkyrimNet web UI’s own validation runs — if the file is bad, the UI reports its usual error (toast notification or alert) and the importer moves on to the next file. Failed files are counted as ✗ failed.

Troubleshooting

  • Firefox: not supported yet — needs showDirectoryPicker. Use Chrome or Edge.
  • Bookmark doesn’t do anything: confirm you opened the SkyrimNet web UI page before clicking the bookmark. The script needs the page’s DOM.
  • “Could not capture file input”: the SkyrimNet UI changed its DOM layout. Open an issue or paste the console log; the selectors will need a tweak.
  • Replace mode skipped a pack that should have been deleted: the importer matches the pack name from the JSON exactly against text on the page. If the existing pack has a different name (e.g. capitalisation, prefix, trailing whitespace), it won’t match. Fix the JSON’s name field or rename the existing pack in the UI first.
  • Imports are slow / timing out: open the userscript in your manager’s editor and bump DELAY_IMPORT from 2500 ms to 4000 or 5000.
